## 一、项目概述

**项目定位**
一款面向毕设展览的互动叙事卡牌小游戏。玩家在模拟微信群聊中，通过选择和出牌来消除针对长辈的谣言，同时维护家庭和谐度。

**核心主张**
"柔性辟谣"——打败谣言不仅需要科学数据，更需要沟通的温度与技巧。

**目标用户**
- 主要：受家庭群谣言困扰的年轻群体
- 次要：在旁观中接受科普的中老年群体

**展示场景**
毕设线下展览，扫码即玩（手机端 H5）

**在线地址**
https://fabulous-dasik-41be94.netlify.app/

**本地离线包**
- 路径：`辟谣防卫战-展览体验版.zip`（123KB）
- 内含：`start.bat` 一键启动脚本，双击即可在本地运行游戏
- 依赖：电脑 + 同一 WiFi 下手机浏览器访问

**APK 打包**
- 技术方案：Capacitor
- 状态：待安装 Android Studio 后实施
- 目标：完全独立的手机安装包，无需任何服务器

## 二、技术与开发策略

**技术栈**
- React + Vite（项目初始化）
- Tailwind CSS v4（快速排版）
- framer-motion（卡牌动画）
- Web Audio API（音效生成）
- Capacitor（APK 打包）

**开发策略**
- 纯前端，无后端依赖
- 所有状态存于内存，不依赖数据库
- CSS 像素风 UI（代码实现，无需图片素材）
- `base: './'` 配置，支持相对路径，构建产物可本地运行

## 三、核心游戏机制

### 3.1 双轨数值系统

```
┌─────────────────────────────────────┐
│  家庭和谐度 HP        谣言污染度     │
│  ████████████░░░     ░░░████████████ │
│      100/100            100/100     │
└─────────────────────────────────────┘
```

- **家庭和谐度（亲情值）**：玩家需要保护，开局 100，归零则输
- **谣言污染度**：玩家需要消除的目标，开局 100，归零则赢

### 3.2 人格系统（6 种代表性人格）

| 编号 | 名称 | MBTI | 抗性说明 |
|:---:|------|------|---------|
| 1 | 固执大爷 | ESTJ | 情绪牌效果 -30% |
| 2 | 焦虑老妈 | ESFJ | 论据牌效果 -30% |
| 3 | 养生专家 | ISFJ | 养生类谣言防御 +20% |
| 4 | 阴谋论者 | INTJ | 所有牌效果 -15% |
| 5 | 佛系邻居 | INFP | 和谐度受损 -20% |
| 6 | 爱孙奶奶 | ESFP | 情绪牌效果 +20% |

### 3.3 卡牌系统

**卡牌类型（3 种）**

| 类型 | 名称 | 伤害 | 治疗 | 代表卡牌 |
|-----|------|-----|------|---------|
| 论据卡 | 理性牌 | 12-22 | 5-10 | 权威数据、专家辟谣、真相图表 |
| 情绪卡 | 情感牌 | 5-12 | 10-18 | 温柔共情、真诚赞美、情感攻势 |
| 特殊牌 | 特殊牌 | 0-10 | 8-18 | 发红包、转移话题、晒娃攻击 |

**每回合 6 张手牌**
- **2 张正确辟谣**（降谣言 + 适度回血）
- **4 张错误辟谣**：
  - 2 张明显错误（降谣言很少 + 扣 10HP 惩罚）
  - 2 张误导性错误（接近正确但实际是错的）

**方案 B 核心机制**

| 选项类型 | 效果 |
|---------|------|
| 正确辟谣 | 降谣言伤害 + 获得治疗，无反击 |
| 明显错误辟谣 | 降谣言 30% 伤害 + 扣 10HP 惩罚，无反击 |
| 误导性错误辟谣 | 降谣言 30% 伤害 + 扣 10HP 惩罚，无反击 |

### 3.4 回合流程

```
第 N 回合
  1. 显示当前状态（双 HP、人格、回合数）
  2. 展示当前谣言
  3. 展示 6 张候选卡牌
  4. 玩家点击选择
  5. 即时计算数值变化
  6. 检查胜负 or 进入下一回合
```

### 3.5 错误弹窗科普机制

当玩家选择错误辟谣时，弹出解释弹窗：

**弹窗内容：**
- 标题：「辟谣失败！」
- 你的选择（红色高亮显示）
- 正确辟谣（绿色高亮显示）
- 解释为什么选错了（黄色标题，正文白色）
- 「我知道了」按钮

**流程：**
```
选择错误卡牌
    ↓
显示弹窗
    ↓
玩家点击「我知道了」
    ↓
执行惩罚（-10HP）
    ↓
进入下一回合
```

### 3.6 谣言去重机制

- 同一局游戏内，已出现的谣言不会重复出现
- 通过 `usedRumors` 数组追踪已用谣言
- `getRandomRumor(exclude)` 支持排除指定谣言

### 3.7 胜负判定

| 结果 | 条件 | 显示 |
|-----|------|-----|
| 胜利 | 谣言污染度先归零 | "辟谣成功！" |
| 失败 | 和谐度先归零 | "亲情破裂…" |

## 四、界面设计

### 4.1 页面结构

```
[开场页] → [人格选择] → [游戏进行] → [结算页]
```

**开场页**
- 游戏标题「辟谣防卫战」
- 视觉填充风格：大字体、居中布局、呼吸动画按钮
- "开始游戏" 按钮

**人格选择页**
- 6 个角色头像卡片（2 行 3 列）
- 显示人格名称、MBTI 标签、emoji 头像

**游戏主界面**
- 顶部：双 HP 条 + 回合数 + 人格信息
- 中部：当前谣言显示区域
- 底部：6 张候选卡牌

**错误弹窗**
- 全屏遮罩（半透明黑色）
- 居中卡片式弹窗
- 像素风边框（警示色）

**结算页**
- 胜负结果
- 坚持回合数
- "再来一局" 按钮

### 4.2 视觉风格

**像素风 + Brutalist（CSS 实现）**
- 深色背景：#1A1A2E
- 霓虹色调：青色(#00D9FF)、红色(#FF6B6B)、黄色(#FFD93D)、绿色(#6BCB77)
- 像素风边框和阴影效果
- 居中大字体标题设计（44px）
- 卡牌动画使用 framer-motion

## 五、音效设计

使用 Web Audio API 生成 8-bit 风格音效：
- 按键音：按钮点击
- 伤害音：谣言受到伤害
- 治疗音：和谐度恢复
- 胜利音：游戏胜利
- 失败音：游戏失败
